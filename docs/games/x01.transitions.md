## 1. State Model (Explicit)

The following state fields are assumed to exist.
These are *derived state*, never directly mutated outside transitions.

### Per Game

* `starting_score`
* `in_strategy` ‚àà {StraightIn, DoubleIn, MasterIn}
* `out_strategy` ‚àà {StraightOut, DoubleOut, MasterOut}
* `legs_to_win`
* `current_leg_index`
* `game_complete`

### Per Leg / Player

* `score`
* `is_in` (bool)
* `legs_won`

### Per Turn

* `turn_active` (bool)
* `turn_start_score`
* `darts_thrown_in_turn` ‚àà {0,1,2,3}
* `current_player`

## 3. Transition Tables

Each table is **orthogonal** and must be applied in order.

---

## Table A ‚Äî Turn Start

| Current State  | Event       | Guard | Result                     |
| -------------- | ----------- | ----- | -------------------------- |
| No active turn | TurnStarted | ‚Äî     | `turn_active = true`       |
| No active turn | TurnStarted | ‚Äî     | `darts_thrown_in_turn = 0` |
| No active turn | TurnStarted | ‚Äî     | `turn_start_score = score` |

**Invalid**

* TurnStarted while `turn_active == true` ‚Üí reject

## Table B ‚Äî DartThrown (General Acceptance)

| State Predicate  | Event      | Guard | Result |
| ---------------- | ---------- | ----- | ------ |
| Game complete    | DartThrown | ‚Äî     | Reject |
| Turn inactive    | DartThrown | ‚Äî     | Reject |
| DartsThrown == 3 | DartThrown | ‚Äî     | Reject |

## Table C ‚Äî In Strategy Resolution

### Applies **only if `is_in == false`**

| In Strategy | DartThrown              | Guard | Result                             |
| ----------- | ----------------------- | ----- | ---------------------------------- |
| Straight In | Any hit                 | ‚Äî     | `is_in = true`, proceed to scoring |
| Double In   | Hit is double           | ‚Äî     | `is_in = true`, proceed to scoring |
| Double In   | Hit not double          | ‚Äî     | No score change                    |
| Master In   | Hit is double or triple | ‚Äî     | `is_in = true`, proceed to scoring |
| Master In   | Hit single              | ‚Äî     | No score change                    |

**Notes**

* Dart counts as thrown regardless of success
* Failure does NOT end the turn

---

## Table D ‚Äî Scoring (Only if `is_in == true`)

Let:
`throw_value = segment √ó multiplier`
`resulting_score = score - throw_value`

| State | DartThrown | Guard                | Result                    |
| ----- | ---------- | -------------------- | ------------------------- |
| In    | DartThrown | resulting_score > 1  | `score = resulting_score` |
| In    | DartThrown | resulting_score < 0  | Bust                      |
| In    | DartThrown | resulting_score == 1 | Bust                      |
| In    | DartThrown | resulting_score == 0 | Proceed to Out Validation |

---

## Table E ‚Äî Out Strategy Validation

Triggered **only when `resulting_score == 0`**

| Out Strategy | DartThrown              | Guard | Result       |
| ------------ | ----------------------- | ----- | ------------ |
| Straight Out | Any hit                 | ‚Äî     | LegCompleted |
| Double Out   | Hit is double           | ‚Äî     | LegCompleted |
| Double Out   | Hit not double          | ‚Äî     | Bust         |
| Master Out   | Hit is double or triple | ‚Äî     | LegCompleted |
| Master Out   | Hit single              | ‚Äî     | Bust         |

## Table F ‚Äî Bust Resolution

A **Bust** is triggered if:

* resulting_score < 0
* resulting_score == 1
* Out strategy validation fails

| State         | Event | Guard | Result                     |
| ------------- | ----- | ----- | -------------------------- |
| Bust occurred | ‚Äî     | ‚Äî     | `score = turn_start_score` |
| Bust occurred | ‚Äî     | ‚Äî     | Force TurnEnded            |

## Table M ‚Äî Round Cap Termination (optional)

Fires **only** when `x01_total_rounds` (per-leg cap) is set. Evaluated on
`TurnEnded` when the turn that just ended belonged to the last competitor of
the round. The cap's sole purpose is to bound per-leg length ‚Äî a natural
checkout during the capped round still wins the leg normally (Table J takes
priority because it fires inside `DartThrown`, before the `TurnEnded` that
would trigger this table).

**Trigger condition**

```
cap_reached = x01_total_rounds != null
           && current_round_in_leg >= x01_total_rounds
           && current_turn_index == competitors.length - 1   // last competitor
```

**Winner selection (no existing winner on the board)**

| Competitor count       | Metric                     | Tie-break       | Outcome on tie-break failure |
| ---------------------- | -------------------------- | --------------- | ---------------------------- |
| 1 (solo)               | ‚Äî                          | ‚Äî               | game ends, winner = `null`   |
| ‚â• 2, clear lowest score | lowest `score` wins       | ‚Äî               | ‚Äî                            |
| ‚â• 2, top two tied       | ‚Äî                          | ‚Äî               | **Ambiguous** ‚Äî UI prompt    |

**Outcomes**

| Situation                                   | LegOutcome signal  | State change                                                                       |
| ------------------------------------------- | ------------------ | ---------------------------------------------------------------------------------- |
| Solo or auto-winner with `legs_won ‚â• legs_to_win` | `gameCompleted` | `is_complete = true`; emit `LegCompleted` + `GameCompleted`                        |
| Auto-winner, more legs remaining            | `legCompleted`     | Increment winner `legs_won`, apply **Table K** reset, emit `LegCompleted`          |
| Multi-player with no auto-winner            | `roundCapReached`  | Persist only `TurnEnded`; notifier sets `pendingCapSelection = true` for UI prompt |

**UI ambiguity resolution**

After `roundCapReached`, the user selects a winner from the cap-selection
dialog. The notifier's `selectCapWinner(competitorId)` emits a synthetic
`LegCompleted` event through the engine's standard path (Table J), so
`legs_won` increments and Table K / Table L handle the subsequent
transition uniformly.

**Invariants**

* The cap never fires during a `DartThrown` ‚Äî only on `TurnEnded`.
* A natural checkout on the last dart of the capped round completes the leg
  before any `TurnEnded` is emitted.
* When `LegOutcome.roundCapReached` is returned, no `LegCompleted` or
  `GameCompleted` event has yet been persisted.

* `0 ‚â§ darts_thrown_in_turn ‚â§ 3`
* `score ‚â• 0`
* Only one active turn at a time
* Bust always ends the turn
* Leg completion can only occur via `resulting_score == 0`
* In-strategy applies only until satisfied

## 5. Notes on Ambiguities (Explicitly Resolved)

The following interpretations are **required** for determinism:

1. A failed **In** dart still consumes a dart
2. Bust always immediately ends the turn
3. Out validation occurs **before** dart count increment is considered terminal
4. Bull (50) counts as a **double** for in/out validation
5. A miss (`segment = 0`) skips Tables C, D, and E entirely. It applies only Table G (dart count increment) and feeds into Table H (turn end condition check). It does not change score, does not satisfy in-strategy, and cannot cause a bust.

These are consistent with standard darts practice and your document.
